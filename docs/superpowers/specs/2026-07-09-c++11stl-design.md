# c++11STL 设计文档

> 从零仿写一个基于 C++11 的 STL 简化实现，作为简历项目。参考 `C:\项目\MyTinySTL`（Alinshans/MyTinySTL）的分层结构，但所有代码从零写、命名空间不同、注释用自己的话写。MyTinySTL 仅作"卡住时对照参考"用，不当抄写源。

## 一、项目定位与目标

- **本质**：学习 STL 底层机制的实战项目，header-only，仿标准库分层。
- **简历价值判定**：能否写进简历取决于能否在面试中讲清每个设计决策（为什么二级配置器、SFINAE 怎么用、红黑树调整规则等），而非代码行数。
- **质量优先**：不赶死线，两个月为参考线而非硬约束。每个模块讲透原理、写扎实、过 review + 过测试才算完成。
- **不做的事**：不新增 STL 之外的功能；不为跑通而注释掉警告或断言；不抄 MyTinySTL 代码（结构可仿、实现自己写）。

## 四、模块依赖与推进顺序

依赖自底向上，下层绝不依赖上层：

```
基础层   type_traits ──┬── iterator
                       └── util ── exceptdef
内存层   alloc ── allocator ── construct ── uninitialized
算法层   algobase ──┬── algo ──┬── heap_algo
                  │          └── set_algo
                  └── algorithm.h (聚合)
容器层   vector / list / deque
         rb_tree ── map / set
         hashtable ── unordered_map / unordered_set
         basic_string ── astring (string = basic_string<char>)
功能层   functional / numeric / stream_iterator
```

**推进顺序（方案 B：纵切 MVP + 补厚）**：

1. 骨架：CMake + 测试框架雏形 + 一个能编译运行的空 `vector_test`
2. **纵切 MVP**：type_traits 最小子集 + 简版 allocator（直接 malloc/free，无内存池）+ vector（构造/析构/push_back/size/operator[]/begin/end）→ vector_test 跑通
3. 补基础层：type_traits 全集 + iterator 完整 + util + exceptdef
4. 补内存层：内存池 alloc + 重构 vector 用正式 allocator + construct/destroy + uninitialized_*
5. 算法层：algobase → algo → heap_algo → set_algo
6. 容器序列：list → deque
7. 关联容器：rb_tree → map/set（最难，预留充足时间，正式写前单开原理专题）
8. 无序容器：hashtable → unordered_map/unordered_set
9. 收尾：functional + numeric + string + 补全测试

**协作节奏**：每个模块 Claude 先讲原理 + 给出该模块要实现什么、关键设计决策，但不直接给完整代码；用户动手写；Claude 读工作目录文件 review：指出 bug、设计问题、与 MyTinySTL/真实 STL 的差异并讲清原因；改到过 review + 过对应 `*_test` 才算完成。

## 五、测试框架设计

- **`TestCase` 基类 + `UnitTest` 单例**：`UnitTest::Run()` 遍历所有 case 依次执行并统计 pass/fail。
- **`TEST(name) { ... }` 宏**：展开成继承 `TestCase` 的类 + 静态实例（构造时自动注册到 `UnitTest` 单例）+ `Run()` 实现。
- **断言宏**（参数顺序 `(实际, 期望)`，与 gtest 一致）：
  - `EXPECT_TRUE(cond)` / `EXPECT_FALSE(cond)`
  - `EXPECT_EQ/NE/LT/LE/GT/GE(v1, v2)`
  - `EXPECT_STREQ/STRNE(s1, s2)`（C 字符串专用）
  - 失败时打印实际值与期望值，用 `<<` 输出
- **简化点**（与 MyTinySTL 的差异）：
  - 第一版无彩色输出（纯文本），后期补 30 行 color.h
  - 第一版无性能测试（MyTinySTL 有 `PERFORMANCE_TEST_ON`），后期作为独立宏补
  - 无 `LARGER_TEST_DATA_ON`，数据量写死在各测试里
  - `main()` 只调 `UnitTest::GetInstance()->Run()`（宏自动注册，比 MyTinySTL 手动列调用更简洁）
- **静态初始化顺序规避**：所有 test.h 只被 test.cpp 一个文件 include，单翻译单元规避静态初始化顺序陷阱。
- 断言宏用 `do { ... } while(0)` 包裹防悬空 else。

## 六、各层详细设计

### 6.1 基础层

**type_traits.h**：
- 纵切 MVP 最小子集：`enable_if`、`is_integral`（手写偏特化，不用宏批量生成）、`remove_cv`。
- 补基础层阶段全集：`is_same`/`is_pointer`/`is_void`/`is_array`/`is_class`/`is_base_of`/`conditional`/`true_type`/`false_type`/`and_`/`or_`/`not_`。
- 面试点：SFINAE（`enable_if<false>` 无 `type` 成员致替换失败跳过重载）；偏特化实现编译期条件。

**iterator.h**：
- 五种 category tag 用继承表达层级（`forward_iterator_tag : input_iterator_tag`），让 `advance` 对 forward 也匹配 input 版本。
- `iterator` 模板结构体（5 个 typedef）。
- `iterator_traits` + 对裸指针偏特化（从 `T*` 萃取 `random_access_iterator_tag`）。
- `distance`/`advance`：category tag 重载编译期分派。
- `reverse_iterator` 适配器。
- 面试点：为什么用 tag 继承重载分派而非虚函数（编译期多态零开销）；`iterator_traits` 对裸指针偏特化的意义。

**util.h**：`move`/`forward`（引用折叠规则 + `remove_reference`）、`swap`（ADL + noexcept）、`pair` + `make_pair`。

**exceptdef.h**：`MYSTL_DEBUG_ASSERT`，调试期断言越界/空指针，Release 走空操作零开销。

### 6.2 内存层

**alloc.h**（二级分配器 + 内存池，补厚阶段才写完整版）：
- 第一级：直接 `malloc`/`free`（>128B）。
- 第二级：内存池 + 16 条 free list（≤128B，按 8 字节对齐到 8/16/.../128 共 16 档）。
- `allocate(n)`/`deallocate(p,n)`/`reallocate(p,old_n,new_n)`，free list 单向链表 O(1) 摘挂。
- 内存池耗尽策略：向系统再要，不够则找更大 free list 借。
- 面试点：为什么二级（小内存碎片+系统调用开销）；free list 单向链表 O(1)；和现代 `std::allocator` 的区别。

**allocator.h**：对 `alloc` 的模板封装，标准 allocator 接口（7 个 typedef + `allocate`/`deallocate`/`construct`/`destroy`/`max_size`/`address` + `rebind`）。分层：内存（alloc）vs 对象（allocator）解耦。

**construct.h**：`construct`（placement new + 完美转发）、`destroy`（单个 + 批量，批量用 trait 分派 trivially destructible 走空操作）。

**uninitialized.h**：`uninitialized_copy`/`fill`/`fill_n`/`move`，trait 分派 trivially copyable 走 `memmove`/`memset`，非平凡走 placement new 逐个构造，异常中途失败回滚（强异常保证）。

**纵切 MVP 阶段简化**：allocator 直接 malloc/free 无内存池；construct/uninitialized 用朴素版本无 trait 分派（trait 全集补齐后再优化）。

**memory.h**：聚合上述四个，对外统一入口。

### 6.3 算法层

**algobase.h**：`max`/`min`/`minmax`（带比较器）、`copy`/`copy_backward`/`copy_n`（category + trait 联合分派）、`fill`/`fill_n`、`equal`、`lexicographical_compare`、`lower_bound`/`upper_bound`（用 `distance`/`advance` 而非 `+`，体现 iterator category 的意义）、`mismatch`。

**algo.h**：
- `sort`：**introsort**（快排递归浅 + 深度过深切堆排保证 O(nlogn) 最坏 + 小区间切插入排序省递归），深度阈值 `2·log n`、区间阈值 16。直接一次到位，不写朴素快排。
- `partial_sort`/`nth_element`（堆应用）。
- `partition`/`stable_partition`。
- `rotate`：第一版三段反转通用版（`reverse(first,mid); reverse(mid,last); reverse(first,last)`），补厚阶段按 category 优化。
- `unique`/`reverse`/`find`/`find_if`/`count`/`for_each`/`binary_search`/`includes`/`merge`/`inplace_merge`。

**heap_algo.h**：`push_heap`/`pop_heap`/`sort_heap`/`make_heap`/`is_heap`，上浮下沉，下标算父子（`i` 父 `(i-1)/2`，子 `2i+1`/`2i+2`），只接受 random_access_iterator。`priority_queue` 底层。

**set_algo.h**：`set_union`/`intersection`/`difference`/`symmetric_difference`/`includes`，双指针归并。

**algorithm.h**：聚合上述全部。依赖：algobase 独立；algo 依赖 algobase + heap_algo；set_algo 依赖 algobase。

### 6.4 容器层

**序列**：
- `vector`：三指针（start/finish/end_of_storage）；`push_back` 扩容（1.5 或 2 倍）走 uninitialized_move + destroy + deallocate，强异常保证（move_if_noexcept 退化）；`insert`/`erase` 注意移动区间方向；`reserve`/`resize`/`shrink_to_fit`；迭代器裸指针。纵切 MVP 只写构造/析构/push_back/size/operator[]/begin/end。
- `list`：双向链表 + 哨兵环形；`node_alloc` 分配节点；`splice` O(1)；list 自己的 `sort`（归并）、`merge`/`unique`/`reverse`。
- `deque`：map（指针数组）+ buffer 双层；迭代器持 `cur`/`first`/`last`/`node`，跨 buffer 跳转；头尾 O(1)；`operator[]` 二级寻址。全 STL 最复杂结构。`stack`/`queue` 适配器包 deque。

**关联**：
- `rb_tree`（全项目最难）：节点 color + parent/left/right + value + header 哨兵环形；插入 BST + 红黑性质调整（叔叔颜色 3 种情况 + 左旋右旋），删除双黑修复；`map` 元素 `pair<const Key,T>` 有 `operator[]`，`set` 元素就是 Key。**正式写前单开红黑树原理专题**。面试点：红黑 vs AVL（牺牲平衡换更少旋转）、5 条性质、header 哨兵简化边界。

**无序**：
- `hashtable`：开链法（桶数组 `vector` + 节点链表）；`hash(K)%bucket_count` 定桶逐个 `==` 查；负载因子超阈值（默认 1.0）rehash；桶数质数（SGI）vs 2 幂（现代）留实现时定。`unordered_map`/`unordered_set` 封装，`unordered_map::operator[]` 同 map。面试点：开链 vs 开放寻址；rehash 时机成本；hash 函数选择（`functional.h` 里 `hash<T>` 特化）。

**字符串**：`basic_string`（模板 + 字符 traits）→ `astring.h` 定义 `using string = basic_string<char>`。第一版**不做 SSO**，朴素堆分配，做核心子集不全文兼容，补厚阶段再考虑 SSO。

### 6.5 功能层

- `functional.h`：`less`/`greater`/`equal_to`/`hash`（各种特化，容器必需）为必做；`function`（type-erasure）和 `bind` 为选做（较难，留最后有时间再做）。
- `numeric.h`：`accumulate`/`partial_sum`/`adjacent_difference`/`iota`。
- `stream_iterator.h`：`istream_iterator`/`ostream_iterator`。

## 七、面试要能回答的关键问题

- 为什么二级配置器？内存池解决什么？free list 为什么 O(1)？
- `vector` 扩容 1.5 或 2 倍为什么？`push_back` 强异常保证怎么实现？`move_if_noexcept` 什么时候退化为 copy？
- SFINAE 怎么用 `enable_if` 拦截非迭代器类型？偏特化怎么实现编译期条件？
- 五种 iterator category 为什么要继承？`iterator_traits` 对裸指针为什么要偏特化？
- `sort` 的 introsort 三件套怎么切换？阈值怎么定？为什么最坏 O(nlogn)？
- `rotate` 为什么不能用 `(first+last)/2`？
- 红黑树 5 条性质？插入后叔叔颜色 3 种情况怎么处理？删除双黑怎么修复？红黑 vs AVL？
- 哈希表开链 vs 开放寻址？rehash 时机和成本？质数桶 vs 2 幂桶？

## 八、保底/冲刺两层目标

- **保底（8 周）**：纵切 MVP + 基础设施补厚 + vector/list/deque + algobase/algo + map/set。到这一步简历已可写，核心面试点全覆盖。
- **冲刺（+2-4 周）**：hashtable + unordered_* + string + functional/numeric，补成全量。即使红黑树卡住也有完整子集兜底，不因最后一公里没到而整个项目报废。
